Switching from Matlab to Python¶

by Nick Cortale

This guide will hopefully ease the transition from Matlab to python. Most of this guide was taken from Jake Vanderplas's guide at:

http://www.astro.washington.edu/users/vanderplas/Astr599/notebooks/

I strongly suggest checking out all his notebooks. They are full of great information. This guide is to be more of a short introduction and hopeully make the transition less daunting.

The main difference between python and matlab is that matlab is like an all inclusive resort. Matlab provides everything that is needed in a complete, polished package. Very few headaches, but there is little interaction with things that are outside of the "matlab area". Things like interfacing with the web are signficantly easier with python.

Python is more like backpacking through Europe (or I would imagine). Many people have done it before and created well documented routes and adventures that work well. You also might get lost on the way and have to create your own path. You also learn a lot on the way. This is why, in my opinion, the python community is so welcoming. They've all been on the same journey.

Downloads

The first thing you will need to do is download the anaconda package manager. This installs everything you will need in the scientific stack --or mostly everything. This can be thought of as installing all the matlab toolboxes. This download will also include the jupyter notebook, which is what this is written in. The download can be found here:

Anaconda : https://store.continuum.io/cshop/anaconda/

Comparing some code¶

Matlab:¶

%some matlab code
x = linspace(0,2*pi);
y = sin(x);
plot(x,y)

Python:¶

In [1]:
#Some python code
import matplotlib.pyplot as plt
import numpy as np
%matplotlib inline

x = np.linspace(0,2*np.pi)
y = np.sin(x)

plt.plot(x,y)
Out[1]:
[<matplotlib.lines.Line2D at 0x10604ad10>]

Okay, so the first obvious difference is the import at the top of the code. Since python has such a huge opensource ecosystem, we have to tell python which packages we wish to utilize. The "matplotlib.pyplot" is our plotting library. and the "%matplotlib inline" is called a magic. Don't worry about it too much. It allows us to have "inline plots".

Also notice the "np." before linspace, pi, and sin. This tells python that we want to use the numpy versions of those functions. For example, there are other implentations of "min" and "max" in numpy than in base python. The same thing goes for "plt.".

Views are not copies¶

Matlab creates copies of everything and this makes it sometimes very inefficient. Python creates a view into an array. This means that it is not an actual copy, but just a pointer to the other array. This is a little confusing until you see it in action

In [56]:
x = np.arange(8)
x_2 = x.reshape(2, 4)


print x
print
print x_2

[0 1 2 3 4 5 6 7]

[[0 1 2 3]
 [4 5 6 7]]
In [57]:
x[0] = 1000
print x
print
print x_2

[1000    1    2    3    4    5    6    7]

[[1000    1    2    3]
 [   4    5    6    7]]

We can see that although we only modified x, x_2 was also changed
